# دليل اختبار المشروع مجاناً (بدون سيرفر مدفوع)

نظراً لأن مشروعك يستخدم قاعدة بيانات **SQLite** (ملف محلي)، فإن رفعه على استضافات سحابية مجانية مثل (Render, Heroku, Vercel) سيواجه مشكلة كبيرة: **سيتم مسح البيانات عند كل إعادة تشغيل للسيرفر** لأن هذه الاستضافات لا تحتفظ بالملفات المخزنة محلياً (Ephemeral Filesystem).

لذلك، أفضل وأسرع حل لعرض المشروع للعميل أو تجربته "أونلاين" دون دفع تكاليف ودون فقدان البيانات هو استخدام تقنية **Nrok** أو **LocalTunnel**.

هذه الأدوات تجعل السيرفر المحلي على جهازك متاحاً عبر رابط إنترنت عام (Public URL) مؤقتاً.

---

## الطريقة الأسهل: استخدام LocalTunnel (مجاني تماماً)

لا تحتاج للتسجيل أو تثبيت برامج معقدة.

### الخطوات:

1. **شغل المشروع محلياً:**
   افتح التيرمينال داخل مجلد المشروع وتأكد أن التطبيق يعمل:
   ```bash
   npm run dev
   ```
   *(تأكد أنه يعمل على المنفذ 5000 أو المنفذ الذي حددته).*

2. **أنشئ الرابط العام:**
   افتح **نافذة تيرمينال جديدة** (اترك الأولى تعمل) واكتب الأمر التالي:
   ```bash
   npx localtunnel --port 5000
   ```

3. **انسخ الرابط:**
   سيظهر لك رابط يشبه: `https://slat-jazzy-fox.loca.lt`.
   هذا الرابط يمكنك إرساله لأي شخص أو فتحه من هاتفك لتجربة التطبيق وكأنه مرفوع على سيرفر حقيقي.

   > **هام جداً:** عند فتح الرابط، سيطلب منك الموقع "Tunnel Password".
   > كلمة المرور هي **عنوان IP العام (Public IP)** الخاص بجهازك.
   >
   > **كيف تحصل عليه؟**
   > 1. افتح الرابط: [https://ipv4.icanhazip.com](https://ipv4.icanhazip.com)
   > 2. انسخ الرقم الذي يظهر لك (مثلاً `102.15.20.200`).
   > 3. الصقه في خانة Tunnel Password واضغط "Click to Submit".

---

## الطريقة البديلة: استخدام Replit (استضافة سحابية)

إذا كنت مصراً على رفع الملفات لسيرفر خارجي وليس جهازك:

1. أنشئ حساباً على [Replit.com](https://replit.com).
2. اختر **Create Repl** واختر **Import from GitHub** (إذا رفعت الكود لـ GitHub) أو ارفع الملفات يدوياً.
3. Replit يدعم SQLite ويحتفظ بالملفات، لذا سيعمل المشروع بشكل جيد للتجربة.
4. اضغط **Run**.

---

## الطريقة الاحترافية (إذا قررت التحويل لقواعد بيانات سحابية)

إذا كنت تريد استخدام استضافة مجانية قوية مثل **Render.com**، يجب عليك تغيير قاعدة البيانات من **SQLite** إلى **PostgreSQL** (استضافة سحابية مثل Neon.tech).
هذا يتطلب تعديلات في الكود (`server/db.ts`)، لكنه الخيار الأفضل إذا كنت تبحث عن استضافة دائمة ومجانية ومستقرة.
